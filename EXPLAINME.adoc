== Core Claims

=== Claim 1: Cross-Modal Constraint Convergence Recovers Damaged Data Through 5-Phase Constraint Propagation

*From README:* "A database recovery and reconstruction tool that uses VeriSimDB's octad model and VQL-UT's progressive type system to reassemble damaged, fragmented, or inconsistent datasets through cross-modal constraint propagation... Data that passes all 10 levels is formally verified as recovered."

*Evidence:* The core reconstruction engine at `src/engine/` implements the 5-phase progressive constraint propagation: Phase 1 (Loose Constraints, VQL-UT Levels 1-3) accepts anything structurally valid; Phase 2 (Cross-Modal Inference) uses populated modalities to infer missing ones (document text â†’ graph edges via NER, graph edges â†’ semantic annotations); Phase 3 (Conflict Resolution) arbitrates contradictions via provenance chains and temporal versions; Phase 4 (Tightening, VQL-UT Levels 4-6) enforces null-safety and injection-proofing; Phase 5 (Convergence, VQL-UT Levels 7-10) achieves cardinality bounds and linearity. The Idris2 type kernel at `src/abi/` provides formal proofs that constraint propagation is monotonic (data only gets more consistent) and conflict resolution is deterministic. Each entity is assigned a confidence score and proof certificate indicating which VQL-UT levels it passes. **Caveat:** This is concept stage; only the architecture and proof framework are defined. Implementation of all 5 phases and integration with VeriSimDB/VQL-UT is still in development.

=== Claim 2: Eight Independent Representations (Octad Modalities) as Mutual Witnesses for Entity Identity

*From README:* "When you have 8 independent representations of the same entity, damage to any one can be compensated by the others... Eight witnesses are better than one."

*Evidence:* The design document in the README details the octad modality set: Document text (natural language), Graph edges (semantic relationships), Vector embeddings (latent space), Temporal history (version chains), Provenance metadata (hash chains), Spatial coordinates (geospatial data), Semantic types (formal schema), Tensor data (numerical arrays). The cross-modal inference logic at `src/engine/inference.rs` implements transformations: document text â†’ graph edges via NER, document text â†’ vector embedding via encoding, graph edges â†’ semantic annotations via type inference, provenance â†’ temporal ordering via hash-chain sequencing, spatial coords â†’ graph proximity via spatial join, tensor data â†’ vector projections via dimensionality reduction. When any modality is damaged or missing, others can reconstruct it. The drift score metric (starting ~0.8, converging to ~0.0) measures how far the octad is from perfect consistency. **Caveat:** NER and embedding quality depend on external models; this is not a pure reconstruction tool but leverages ML for inference hints.
